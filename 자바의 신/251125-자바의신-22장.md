# 컬렉션 (List)

## List란?

- `java.util.List<E>`는 인터페이스이다
- 순서를 유지하고, 중복을 허용하고, 인덱스로 접근하며, 제네릭 타입을 쓴다

## List의 구현체

### ArrayList

- 내부는 배열로 구현
- 스레드에 안전하지 않음
- 장점
    - index로 접근하는 `get`, `set`이 O(1)
    - 맨 뒤에 `add()` 하는 것도 평균적으로 빠름 (배열 확장할 때만 느림)
        - 배열이 꽉 차 있으면, 더 큰 배열을 새로 만들어 복사
- 단점
    - 중간의 요소를 삽입/삭제 시 뒤쪽 요소들을 한 칸씩 밀거나 당겨야 해서 O(n) 발생

### LinkedList

- 내부는 양방향 연결 리스트
- 스레드에 안전하지 않음
- 장점
    - 리스트 중간에서 삽입/삭제 연산이 노드 연결만 바꾸면 되니 이론적으로 유리하다
- 단점
    - 인덱스 접근 시 앞에서부터 차례로 따라가야해서 느림 O(n)

### Vector

- ArrayList와 거의 동일한 구조
- 모든 메소드가 synchronized 되어 있어서 스레드 안전
- 장점
    - 스레드 안전
    - 인덱스 접근 빠름 -> O(1)
- 단점
    - 모든 메소드가 synchronized로 동시성 제어가 너무 심해 성능 저하가 심하다

### Stack

- LIFO 자료구조 제공
- 내부적으로 Vector를 상속한 클래스
- 장점
    - 스레드 안전 (Vector 상속이므로)
    - 내부는 배열 기반으로 pop/push 빠름
- 단점
    - Vector 기반이라 성능이 안좋음
    - Deque를 쓰는 것이 훨씬 좋음
