# java.lang 패키지

## 숫자를 처리하는 클래스

- 기본 자료형을 객체로 처리해야 할 때 사용할 수 있는 클래스는 다음과 같다
    - `Byte`, `Short`, `Integer`, `Long`, `Float`, `Double`, `Character`, `Boolean`
- `Character`와 `Boolean`을 제외한 클래스는 `Wrapper` 클래스라고 부르며, 모두 `Number`라는 `abstract` 클래스를 확장한다
- 또한 이들은 자바 컴파일러에서 자동으로 형 변환을 해주기 때문에 기본 자료형처럼 사용할 수 있다
- `Character` 클래스를 제외하고는 `parse타입이름()`, `valueOf()`라는 공통적인 메소드를 제공한다
    - 이 둘은 `static` 메소드이며, `parse타입이름()`은 기본 자료형을 리턴하고, `valueOf()`는 참조 자료형을 리턴
- Boolean 클래스를 제외하고는 모두 `MAX_VALUE`, `MIN_VALUE`라는 상수를 가지고 있다

### Number 클래스란?

- 정수/실수 타입마다, 클래스가 따로 존재하면 공통적으로 처리하기 불편하기 때문에 묶은 클래스
- 모든 수를 기본형으로 꺼낼 수 있도록 추상 메소드를 제공한다

```java
public abstract int intValue();
public abstract long longValue();
public abstract float floatValue();
public abstract double doubleValue();
public abstract byte byteValue();
public abstract short shortValue();
```

## System 클래스

- 시스템에 대한 정보를 확인하는 클래스이다.
    - 시스템 속성(Property)값 관리
    - 시스템 환경(Environment)값 조회
    - GC 수행
    - JVM 종료
    - 현재 시간 조회
    - 기타 관리용 메소드

### 시스템 속성 값 관리

- `Properties` 클래스는 java.util 패키지에 속하며, `HashTable`의 상속을 받았다
- 자바 프로그램을 실행하면 `Properties` 객체가 생성되며, 그 값은 언제/어디서든지 같은 JVM 내에서는 꺼내서 사용할 수 있다
- `Properties` 클래스는 추가/변경이 가능하다

### 시스템 환경 조회

- 환경값 env는 변경하지 못하고 읽기만 가능하다
- 이 값들은 대부분 OS나 장비와 관련된 것들이다

### GC 수행

- 자바에서 메모리 처리는 개발자가 별도로 처리하지 않지만 `System.gc()`라는 메소드를 호출하여 GC를 명시적으로 처리할 수 있다
- Object 클래스에 선언되어 있는 `finalize()`를 명시적으로 수행하도록 하는 `runFinalization()` 메소드가 있다
- 이 두 메소드를 실행하면 시스템은 하려던 일들을 멈추고 이 작업을 실행한다

#### finalization 이란?

- 옛날 자바에서 객체가 GC 대상이 될 때, 그 객체가 `finalize()` 메소드를 오버라이드 해뒀다면, 바로 메모리에서 지우지 않고, finalize 큐에 넣고 나중에 JVM 내부의 finalizer 전용 스레드가 그 객체의 `finalize()`를 호출해 그때서야 진짜 메모리에서 제거한다
- `System.runFinalization()`이 하는 일은 대기 중인(finalization pending) 객체들의 `finalize()`를 지금 실행하라고 JVM에 요청하는 역할을 수행한다
- `finalize()`는 자바 9부터 Deprecated 되었다

### 현재 시간 조회

- `currentTimeMillis()`와 `nanoTime()` 메소드가 있다
- `currentTimeMillis()`
    - 현재 시간을 나타낼 때 유용
- `nanoTime()`
    - 메소드 시간 측정을 위해서 만들어짐

### System.out

- 다음의 코드처럼 `null`을 출력하는 메소드가 있다고 하자

```java
public void pringNull() {
    Object obj = null;
    System.out.println(obj);
    System.out.println(obj + " is object's value");
}
```

- 이 메소드를 호출하면 다음과 같이 출력된다
- 그 이유는 컴파일러에서 이 더하기 문장을 `StringBuilder`로 변환해서 처리하기 때문이다

```
null
null is object's value
```

- 하지만 다음과 같이 toString을 사용하면 `NullPointerException`이 발생한다
- 객체를 출력할 때는 `toSting()`보다 `valueOf()`를 사용하는 것이 안전하다

```java
public void pringNull() {
    Object obj = null;
    System.out.println(obj.toString());
}
```
