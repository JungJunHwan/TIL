# 상속

## 상속이란

- 상속이란 자식 클래스에서 부모 클래스를 확장한다는 말이다
- 확장을 하면 부모 클래스에 선언되어 있는 `public` 및 `protected`로 선언되어 있는 모든 변수와 메소드를 내가 갖고 있는 것처럼 사용할 수 있다
- 자바는 다중 상속이 안된다

## Overriding

- 자식 클래스에서 부모 클래스에 있는 메소드와 동일하게 선언하는 것
- 리턴 타입, 메소드 이름, 매개 변수 타입, 및 개수가 전부 동일해야 한다 (= 동일한 시그니처를 갖는다)
- 오버라이딩 시 자식 클래스가 부모 클래스이 접근 제어자보다 확대 하는건 가능하지만, 축소 시키는건 불가능하다

## 형 변환

- 자식이 부모 생성자를 사용할 수 없는 이유
    1. 부모 객체에는 자식의 필드, 메소드 등이 없기 때문
    2. 객체 생성 순서는 부모 -> 자식 순으로 강제되어 있음
        - 자식 객체 생성 시 내부 단계
            1. 부모 생성자 실행
            2. 자식 생성자 실행
            3. 자식 필드 초기화
            4. 자식 초기화 블록
        - 만약 자식이 부모를 먼저 new로 만들려고 하면
            1. 부모 객체만 생성
            2. 자식 부분 없음
            3. 자식 객체 자체가 존재하지 않음

- 부모 클래스가 자식 생성자를 호출해도 되는 이유
    1. 자식 객체 생성 과정에 부모가 포함되기 때문
        - 객체 생성 순서
            1. 자식 객체 생성 시 부모 생성자가 먼저 호출됨
            2. 부모는 자신의 필드 초기화만 하면 된다
            3. 실제 자식 부분은 자식 생성자에서 마저 초기화 된다

- 부모 클래스가 자식 클래스의 생성자로 객체를 생성하면, 실제 생성된 객체는 자식 클래스의 형태를 가진다

## 다형성

- 하나의 부모로부터 다양한 자식 클래스를 생성하고, 부모 클래스로 선언한 객체를 자식 클래스의 생성자로 생성할 때, 실제 호출되는 것은 원래 객체(자식 클래스로 생성한 객체)에 있는 메소드가 호출된다는 것이 다형성이다
