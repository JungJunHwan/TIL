# 2장 자바와 절차적/구조적 프로그래밍

## 자바에서는 왜 goto를 사용하지 못하게 한 것일까

- got는 쉽게 스파게티 코드(엉킨 코드)를 만든다
- goto를 사용하게 되면 프로그램의 실행 순서를 이리저리 이동할 수 있게 된다
- goto를 사용하면 지금 이 코드가 어디서 왔고, 끝나면 어디로 가는지 파악하기 힘들다
- 유지보수, 디버깅, 협업에 안좋다
    - 사람마다 스타일이 다르기 때문
- 자바는 구조적 프로그래밍, 가독성, 안전성을 목표로 하기 때문에 사용하지 못하게 함

## 함수(Function)과 메소드(Method)의 차이

- 전혀 다르지 않지만, 함수는 클래스나 객체와 아무 상관이 없지만, 메소드는 반드시 클래스 안에 존재해야 한다

## main() 메소드가 실행되기 전 JVM에서 수행하는 전처리 작업

1. JVM 실행
2. 클래스 로더가 main 클래스 로딩
3. Linking
    - 검증
    - 준비 (static 변수 기본값)
    - 해석 (심볼릭 -> 실제 주소)
4. 클래스 초기화
    - static 변수 초기값 실제 할당
    - static 블록 실행
    - 부모 클래스부터 순서대로
5. JVM 메모리 구조 준비 (Heap, Stack 등)
6. main 스레드 생성
7. main() 실행 시작
