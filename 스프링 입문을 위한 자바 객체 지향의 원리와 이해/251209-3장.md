# 3장 자바와 객체 지향

## 객체 지향의 4대 특성 - 캡!상추다

- 캡슐화 (정보 은닉)
    - 데이터(필드)와 기능(메소드)를 하나로 묶고, 외부에서 직접 접근하지 못하게 보호하는 것
    - 객체 내부 상태를 보호
    - 내부 구현을 숨겨 변경 시 외부 코드 영향 최소화
- 상속 (재사용)
    - 기존 클래스의 속성과 기능을 재사용하고, 필요에 따라 기능을 확장할 수 있도록 하는 구조
    - 코드 재사용성 증가
    - 공통 기능을 부모 클래스로 올려 중복 최소화
    - 계층 구조를 만들어 확장에 유리하게 함
- 추상화 (모델링)
    - 핵심적인 특징만 뽑아서 모델링하는 것
    - 구체적인 것을 분해해서 관심 영역에 있는 특성만 가지고 재조합하는 것
    - 복잡한 시스템을 단순하게 이해할 수 있도록
    - 공통점을 기준으로 객체를 정의하여 코드 구조를 명확히
- 다형성 (사용 편의)
    - 같은 메소드 호출이지만, 실제 어떤 객체냐에 따라 다른 동작을 수행하는 것
    - 확장성 증가
    - 동일한 인터페이스로 다양한 구현을 처리

## 클래스와 객체

클래스는 구분이며 객체는 실제 존재이다
- 클래스 : 분류, 집합과 같은 속성과 기능을 가진 객체를 총칭하는 개념
- 객체 : 세상에 존재하는 유일무이한 사물

## 상속

-  상속은 "is-a" 관계라고 많이 설명되지만, 더 자세히는 "is a kind of" 관계이다

## 다중 상속과 자바

- 왜 자바는 다중 상속을 지원하지 않을까
- 인어는 사람과 물고기를 상속한다고 해보자
- 이때 사람과 물고기 모두 수영할 수 있는데, 인어는 사람처럼 팔다리를 저어 수영할까 아니면 물고기 처럼 지느러미로 헤엄쳐야 할까
- 이를 다중 상속의 다이아몬드 문제라고 한다
    - 클래스가 여러 부모클래스를 상속할 때 어떤 부모로부터 어떤 메소드/필드를 상속받아야 하는지 모호해지는 문제
    - 다이아몬드 문제가 위험한 이유
        - 메소드 충돌
        - 필드 중복
        - 유지보수 어려움

## 인터페이스

- 인터페이스는 "be able to", 즉 "무엇을 할 수 있는" 이라는 표현 형태로 만드는 것이 좋다
